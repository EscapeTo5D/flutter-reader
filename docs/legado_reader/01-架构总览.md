# 第1章 架构总览

## 1.1 整体架构分层

文本阅读器采用 **MVVM + 全局单例** 的混合架构，分为五层：

```
┌──────────────────────────────────────────────────────────────────┐
│                       UI 层 (Activity/View)                       │
│  ReadBookActivity ←→ ReadView ←→ PageView ←→ ContentTextView     │
├──────────────────────────────────────────────────────────────────┤
│                    ViewModel 层 (数据处理)                         │
│                       ReadBookViewModel                           │
├──────────────────────────────────────────────────────────────────┤
│                     Model 层 (状态管理)                            │
│                ReadBook (全局单例) + ContentProcessor              │
├──────────────────────────────────────────────────────────────────┤
│                    排版引擎层 (布局计算)                            │
│           ChapterProvider ←→ TextChapterLayout                    │
├──────────────────────────────────────────────────────────────────┤
│                     数据层 (存储/网络)                             │
│            appDb + WebBook/LocalBook + BookHelp                   │
└──────────────────────────────────────────────────────────────────┘
```

  └── ReadBook (全局状态单例)

## 1.4 核心数据流

```
书籍数据 (Book/BookChapter)
  → 内容获取 (BookHelp/CacheBook/WebBook/LocalBook)
    → 内容处理 (ContentProcessor: 替换规则、标题处理)
      → 排版 (TextChapterLayout: 分行、分页)
        → 页面数据 (TextPage/TextLine/Column)
          → 渲染 (ContentTextView: Canvas绘制)
```

## 1.5 页面渲染流程概览

```
1. 用户打开书籍
   ReadBookActivity.onPostCreate()
     → ReadBookViewModel.initData()
       → ReadBook.resetData(book)
       → ReadBook.loadContent(resetPageOffset = true)

2. 内容加载 (IO线程)
   ReadBook.loadContent(index)
     → BookHelp.getContent() 查本地缓存
       → 命中: contentLoadFinish()
       → 未命中: CacheBook.download()

3. 内容处理 (IO线程)
   ReadBook.contentLoadFinish()
     → ContentProcessor.getContent()      // 应用替换规则
     → ChapterProvider.getTextChapterAsync()
       → TextChapter.createLayout()       // 启动异步排版

4. 异步排版 (IO线程)
   TextChapterLayout.getTextChapter()
     → 遍历内容段落
     → setTypeText() / setTypeImage() / setTypeHtml()
       → StaticLayout 或 ZhLayout 计算行布局
       → 逐字符生成 TextColumn
       → durY 超过 visibleHeight → prepareNextPageIfNeed()
         → channel.trySend(textPage)      // 通知UI线程

5. UI更新 (Main线程)
   ReadBook 监听 layoutChannel:
     → 设置 curTextChapter = textChapter
     → callBack.upContent()
       → ReadView.upContent()
         → PageView.setContent(textPage)
           → ContentTextView.setContent()
             → invalidate()

6. 文本绘制 (Main线程)
   ContentTextView.onDraw(canvas)
     → drawPage(canvas)
       → textPage.draw(canvas, relativeOffset)
         → 遍历 TextLine → line.draw(canvas)
           → 遍历 Column → column.draw(canvas)
```

## 1.6 三页缓冲策略

阅读器维护三个 TextChapter 缓冲区，实现无缝翻页：

```
prevTextChapter ← curTextChapter → nextTextChapter
   (上一章)         (当前章)          (下一章)
```

翻章时旋转缓冲区（零拷贝）：

```kotlin
// 下一章
fun moveToNextChapter() {
    prevTextChapter = curTextChapter      // 当前章 → 上一章
    curTextChapter = nextTextChapter      // 下一章 → 当前章
    nextTextChapter = null                // 清空下一章
    loadContent(durChapterIndex + 1)      // 异步加载新的下一章
}
```

## 1.7 Channel 流式排版

TextChapterLayout 通过 `Channel<TextPage>(UNLIMITED)` 逐页推送排版结果：

```
TextChapterLayout (IO线程)          ReadBook (Main线程)
  │                                    │
  ├─ 排版第1页完成                       │
  ├─ channel.trySend(page1) ──────────→ │ 收到page1, 通知UI
  ├─ 排版第2页完成                       │
  ├─ channel.trySend(page2) ──────────→ │ 收到page2, 通知UI
  ├─ ...                               │
  ├─ 排版最后一页完成                     │
  ├─ channel.close()                   │
  └─ isCompleted = true                │ 标记排版完成
```

这种设计使得UI可以 **渐进式渲染**——排好一页就显示一页，不必等待整章排版完成。
